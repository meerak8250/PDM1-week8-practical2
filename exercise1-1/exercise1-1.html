Your sketch:

Instructions

The assets folder contains a .txt file called imdb_labelled.txt. This file contains part of a dataset commonly used to train machine learning models to detect the sentiment of a text. Each line of the file contains a user review of a film from imdb.com followed by a number, either 0 or 1. The number is the "label" of the review. 0 means the review is negative, 1 means the review is positive.

Use the loadStrings() function to read the contents of the file into an array. You may find it helpful to test that the file is read by printing a line from the file to the console.

Next, display a random review (just the text, not the label), whenever the user presses a key. It should be possible to visually distinguish positive reviews from negative reviews. For example, you could use different colours or fonts for each category of review.

A note on using split() for this exercise

To separate the review label from the text, you can use the split() function. Unlike the examples shown in class, the review text and label are separated by a tab. To split on a tab, use the tab escape sequence, \t. E.g.

let parts = split(someString, "\t");

After splitting a line of text, you will need to workout whether the review is positive or negative based on the value of parts[1]. Remember that parts[1] will be a string, either "1" or "0".

Expected output:

Selecting a random index

The floor() function will be useful when picking a random review. Remember that the random() function always returns a float but array indices are always integers. You can use floor() to round a random float to the nearest integer less than or equal to the float.

Parsing the text from the file

Once the file has been read to an array using loadStrings(), there are several ways to achieve the expected output using only concepts covered in the module so far. Here are two possibilities:

  • You could leave the array containing the raw reviews as it is. Then, when a random review is selected for display, use split() to separate the review text and the label. You could then determine the styling for the text based on the review label.
  • Another (better) approach is to write a custom Review class with a property for the review text and a property for the review sentiment. The class constructor can separate the review text and label. In setup(), you can convert each element in the array storing the raw strings read from the file to a Review object. You can then use the randomly selected Review's own properties to figure out how to display it.
